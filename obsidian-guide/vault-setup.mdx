## Understanding Vaults

A **vault** in Obsidian is simply a folder on your computer containing all your notes (`.md` files) and configuration. Think of it as your personal knowledge base.

<Info>
  **Key Insight:** Because vaults are just folders, you can back them up, sync them, or share them like any other files on your computer.
</Info>

## Creating Your First Vault

<Steps>
  <Step title="Open Obsidian">
    Launch Obsidian from your applications
  </Step>
  <Step title="Create New Vault">
    Click "Create new vault" on the start screen
  </Step>
  <Step title="Name Your Vault">
    Choose a descriptive name:
    - "Personal Knowledge Base"
    - "Family Research"
    - "Project Documentation"
    - "Legal Case Prep"
  </Step>
  <Step title="Choose Location">
    Select where to store your vault (we recommend your Documents folder)
  </Step>
</Steps>

## Creating Your Folder Structure

<Steps>
  <Step title="Open File Explorer">
    In Obsidian, look at the left sidebar - this is your file explorer
  </Step>
  <Step title="Create Folders">
    Right-click in the file explorer and select "New folder"
    
    Create the folders from your chosen structure above
  </Step>
  <Step title="Create Your First Note">
    Right-click on a folder and select "New note"
    
    Or use the keyboard shortcut: `Cmd/Ctrl + N`
  </Step>
</Steps>

## Vault Management Tips

### Naming Conventions

<Tip>
  **File naming best practices:**
  - Use descriptive names: "Meeting Notes 2024-01-15" not "notes1"
  - Use consistent date formats: YYYY-MM-DD
  - Avoid special characters: / \ : * ? " < > |
  - Use spaces or hyphens, be consistent
</Tip>

### Tags vs Folders

<Tabs>
  <Tab title="Folders">
    **Use folders for:**
    - High-level organization
    - Grouping related content
    - Projects or topics
    
    Example: `/Projects/Website/` contains all website-related notes
  </Tab>
  <Tab title="Tags">
    **Use tags for:**
    - Cross-cutting themes
    - Status indicators
    - Quick filtering
    
    Example: `#urgent`, `#research`, `#actionable`
  </Tab>
</Tabs>

<Info>
  **Pro Tip:** Use a combination! Folders for structure, tags for flexible categorization.
</Info>

## Backup Strategy

<Warning>
  **Important:** Your vault contains important information. Always keep backups!
</Warning>

### Option 1: Cloud Sync (Easiest)

<CardGroup cols={3}>
  <Card title="Obsidian Sync" icon="cloud">
    Official paid service
    
    $10/month, automatic, encrypted
  </Card>
  <Card title="iCloud / OneDrive" icon="cloud-arrow-up">
    Free option
    
    Put vault in synced folder
  </Card>
  <Card title="Dropbox / Google Drive" icon="folder">
    Free option
    
    Works but may have conflicts
  </Card>
</CardGroup>

### Option 2: Git (Advanced)

For tech-savvy users, use Git for version control:

```bash
cd /path/to/vault
git init
git add .
git commit -m "Initial commit"
```

<Tip>
  The **Obsidian Git** plugin automates this process. Great for programmers!
</Tip>

### Option 3: Manual Backup

Simply copy your vault folder regularly:

<Steps>
  <Step title="Find Vault Location">
    Settings â†’ About â†’ Show vault folder
  </Step>
  <Step title="Copy Folder">
    Copy entire vault folder to external drive or cloud storage
  </Step>
  <Step title="Schedule Regular Backups">
    Set a calendar reminder for weekly/monthly backups
  </Step>
</Steps>

## Multiple Vaults

You can create multiple vaults for different purposes:

<AccordionGroup>
  <Accordion title="When to use multiple vaults">
    - Separate work and personal
    - Different privacy levels
    - Distinct projects that shouldn't mix
    - Testing plugins without affecting main vault
  </Accordion>
  <Accordion title="How to switch vaults">
    1. Click vault name in bottom left
    2. Select "Open another vault"
    3. Choose from existing or create new
  </Accordion>
</AccordionGroup>
